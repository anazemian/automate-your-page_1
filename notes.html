Important Concepts and Languages

Lesson 1: Intro to 'serious' programming

Computer

Most machines like a toaster, or an oven, are designed to do limited things. Unless you physically alter the object, they can only toast bread, or bake a cake, with very limited options such as higher or lower heat.

Computers however, are different in that they can be programmed to do anything we want, so long as we can write a program that specifies a very specific set of instructions.

Programming Language

Programming language is what programmers use to tell a computer what to do. Python is an example of a language.

Python

Python is a programming language. When you write code in Python and then 'run' that code, a Python Interpreter converts the code you wrote as a set of instructions that the computer then understands and executes.

Grammar

Just as human languages have proper grammer, so does Python, or any programming language. Grammar is specification of what is correct and incorrect.

In English for example, people can often make sense of sentences that are not techincally correct, but computers cannot do the same. This mean that we must use proper programming grammar, or write code that is exactly correct according to the interpreter, otherwise the code will not run.

Python Expressions

A Python 'expression' is a legal Python statement. For example: print 1 + 1 is a valide expression, but print 1 + is not, because there is no number at the end.

Lesson 2: Variables

Variables

Variables provide a way for programmers to give a name to a value. For instance, if variable_numbers is a variable with the value of 2 then the following code would print out 4

print variable_number + variable_number

What does it mean to assign value and how do I do it?

We can assign any value to any varible so long as it is the correct syntax and the interpreter can understand the code. For instance we can assign the value of 2 to the variable variable_number by writing the following line of code:

variable_number = 2

We can even change that value by re-assigning it to a different value later, in turn making the coding process easier since we are only changing one value, rather than an entire line of code.

What does the equals ( = ) symbol mean?

In the line 5 + 5 = 10 the equals symbol means exactly that, equals. However in the line of code variable_number = 5 the equals symbol means assigned to; think of it as an arrow rather than an equals.

How are variables usful?

Variables are useful in many ways:

  1. They improve code readability by using names that make sense to people.
  2. They give us a way to store values of important data
  3. They allow for quicker coding. We can change the value of a variable to change numerous lines of code at once, vs changing each line of code one by one
Whats the difference?

In Python 2 is a number, while "2" is a string. The code 2 + 2 would print 4 while the code "2" + "2" would print "22".

Lesson 3: Functions

What is a function?

A function is something that takes an input, does somthing to that input, then produces an output. For example, a function named square may take a number as an input and produce that number squared as an output.

Whats the difference between Making and Using a function?

Functions are made by lines of code starting with the key word def and then giving a function name followed by the parameters in parentheses. These parameters will eventually be replaced by actual values when the function is called.

In the 'body' of the function, we write the code that specifies what do do with the input paramters. i.e:

def square(x):
c = x * x
return c

To use a function, we write the name of the function followed by the value we assing to the parameters in the parentheses:
print square(4)
>>>16

How do functions help avoid repetion?

Functions are tools that can be created and reused forever. Once a function has been defined, it never has to be re-defined!

What if there is no RETURN statement?

The return keyword tells Python exactly what the function needs to produce as the output. If there is no return, then the output will just produce 'None':

def add_two(x):
answer = x +

new_number = add_two(7)
print new_number
>>>None
Concept of Input to Functions to Output

Simply put, when you enter something into a computer, or calculator, or microwave, it takes the command you enter, and gives you an output once it has completed a function. In programming, this can be data, strings, lists, etc. Every line of code after the defining function must be indented in the proper block to ensure you get back the correct output. For example:

def is_friend(name):
return name[0]=='A' or name[0]=='D'

I would get the output:

print is_friend('Amir')
print is_friend('Dani')
>>True
>>True

These two print statements would output True, because my code asks to return the names that begin with either an uppercase A or D at index 0. If I had input the name Mark, the output would be False as it does not start with either letter we call for at index 0.

Using a function

You can set procedures by using a def command at the beginning and it will not be used unless it is called upon later in the code. It will lay dormant until a command is set to run that procedure and return its value. To se a procedure, the basic code would be to give it a name like simple_code, and then you would have your input parameter(s) with in parentheses after the name of the code, such as simple_code(n).

There can be as many inputs as needed, so long as they are separated by commas, and later give the correct amount of inputs in your code.

If you are defining a procedure you put def in front of it and a colon at the end, and on the next line you put the function(s) you want to run. It would end up looking like this.

def simple_code(n1,n2):
return n1+n2
print simple_code(3,5)

The result or output of this function would be 8. It is important to remember that there must be an output for the function. If there is no return statement, as we stated before, nothing will come back.

Using the IF function

To use the if function, you would type if <expression>  and then below you would have the <block> which will define what code is run if the expression is TRUE. If the expression result is FALSE then it will skip this line of code.

With the IF function you can also define and ELSE statement. This means that if your IF statement runs FALSE, it will then go to the ELSE statement and run the line of code there. The ELSE must be on the same indent as the IF.

Here is an example of IF and ELSE:

def bigger(num_1,num_2):
    if num_1 > num_2:
        return num_1
    else:
        if num_1 < num_2:
            return num_2
        else:
            if num_1 == num_2:
                return num_1

This code will always return the greater number of the two inputs.

If you are telling the code to return a True or False then you must capitalize as shown below:

def is_friend(name):
    if name[0] == 'D':
        return True
    else:
        return False
IF and OR

When IF or OR is called as a function to evaluate two or more expressions, it will stop at the first TRUE expression. For example, if you said, 'Am I 30 or 31', the return statement would be TRUE because it would never actually evaluate if you were 31 or not as it knows at least on of the statements is already true. If you were 31 and asked the same question it would evaluate that 30 is false and still return true.

If you need to test both expressions, you may need to just put False in the first to ensure no error in the second as it will not pass the first expression if it returns a true value. It would then have to evaluate the second expression and state if there is an error.

Loops

A loop is just what is sounds like, a loop of an action that just repeates. We start with the WHILE loop.

The WHILE loop starts with an expression that will validate the running of the loop followed by a colon. This is called a test expression. Inside of that we have the block that will run as long as the test expression is true. The difference between this and the IF command, is that once it runs the command will go back to check if the test expression is still true. If it is, the block of code will run again and again and again until the test expression returns a false value.

i = 0
while i < 10
    print i
    i = i + 1

This results in the code printing from 0 to 9. Once it gets to 10 it is no longer less than 10, as stated in the code, so the code loop will no longer run. An infinit loop is just like it sounds, it will never find a false value so it will never stop.

Break Statement

A break statement is a way to stop the loop even when the loop is still returning TRUE. You could stick this in an IF command in the WHILE loop which will check to see if there is a reason to break the process or let it continue. If this IF statement is TRUE and you break the loop, you will then continue the code after the while loop. If the IF statement is false, it will continue to run the code and the loop and will check the IF statement the next time around.

Lesson 4: Lists

Lists

The list type is a container that holds a number of other objects, in a given order. The list type implements the sequence protocol, and also allows you to add and remove objects from the sequence.

Nested Lists

Simply put, a nested list, is a list within a list. Again lists can contain any set of expressions. We could have:

mixed_up = ['apple', 3, 'brown']

double_mix = ['apple' 3 'brown'['music', 'television']]

The above example shows a list with various expressions, and a nested list, a list within a list. A better example of a nested list would be:

beatles = [['John', 1940]]
                  ['Paul', 1942]
                  ['George', 1943]
                  ['Ringo', 1940]]

Each element of this nested list is the name of a Beatle and the year they were born. From here we can call the info we need easily because of the nested list.

print beatles[3][0]
>>Ringo
print beatles[3][1]
>>1940

The [3] is the last element in the nest and the [0] or [1] is the object in that portion of the list.

Mutations

Another difference between lists and strings is Mutation. We can mutate the position value in a list such as:

p = ['h','e','l','l','o']
p[0] = 'y'

Now instead of printing 'hello' our code will print out 'yello' because we mutated the list so that at index 0 p is now 'y' rather than rewriting the entire code.

Aliasing

Just like with names for humans, Aliasing refers to two names being used for the same thing. For example, James Bond is also known as 007. So if something were to happen to James Bond, it also happens to 007. The same goes for lists.

spy=[0,0,7]
agent=spy
print agent
>>[0,0,7]
print spy
>>[0,0,7]

If we change the second value in spy it also changes the second value in agent because agent and spy both refer to the same list.

List Operations

Lists are very powerful and with mutations and aliasing, and other list operations, we can build complex data structures. Here are a few examples of other list operations.

  1. Append
  2. Plus
  3. Len

Append let's add a new element to the end of the list without creating a new list. Plus lets us do just as it sounds, add two lists together. It works very similar to the concatination of two strings And finally, the len operator. Len tells us the length of the object we want to know the length of. Len works for both lists and strings. The output for len is the number of elements in the input. Here are some examples of each.

APPEND

stooges['Moe','Larry','Curly']
stooges.append('Shemp')
stooges['Moe','Larry','Curly','Shemp']


PLUS

[0,1] + [2,3] = [0,1,2,3]


LEN

len([0,1])
>>2
len(['Udacity'])
>>7
len(['a','b',['c']])
>>2

If you have a list within a list, len will only output the outer list. So in the last example above, the output is still 2 because len will not recognize the second list inside the first. Also, unlike most indexes, len begins at 1, not 0.

Loops on lists

There are four ways we can use loops with lists; FOR loops, WHILE loops, and two versions of indexing. Here I will show examples of each.

def measure_udacity(U):
      count = 0
      for e in U:
          if e[0] == 'U'
              count = count + 1
      return count

The above code is a FOR loop and will return the number of the paramter listed next to the defined variable. So if we are looking for names that beging with an uppercase U, the code will return the number of instances the U appears:

print measure_udacity(['Umbrella','John','Uncle'])
>>2
print measure_udacity(['Hank','Walt','Jesse'])
>>0

Next we will look at an example of a while loop with lists.

def find_element(p,t):
      i=0
      while i<len(p):
            if p[i]==t:
                  return i
            i = i+1
      return - 1

In the above example we used a WHILE loop that would output a numerical value that shows a match at the index:

print find_element(['alpha','beta'],'beta')
>>2
print find_element([1,2],3)
>>-1

The output of 2 is that at index position 2 there is a match. In the second print statment there is no match so our output is -1 based on our return statement.

Finally we will look at the index method. if the <value> is in the <list> it reurns the first position where the <value> is found. Otherwise it produces an error. the 'in' operator changes the output to a tru false statement. We can use the index function in various ways as seen below. Both will have the same result.

def find_element(p,t):
      if t in p:
            return p.index(t)
      else:
            return - 1


OR


def find_element(p,t):
      if t not in p:
            return -1
      return p.index(t)

Both of these codes will have the same output as our while loop above. They will print out the position of where there is a match in the nest and the string.
